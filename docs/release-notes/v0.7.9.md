# v0.7.9 Release Notes

## Overview

This release focuses on enhancing AI provider support with DeepInfra integration, improving privacy with owner-only library access, and streamlining deployment with automated Docker builds.

## New Features

### DeepInfra AI Provider Integration
- Added full support for DeepInfra as an AI provider
- Integration with Qwen and DeepSeek models for cost-effective AI processing
- Automatic model selection and configuration in the admin interface

### Owner-Only AI Library Access
- Implemented privacy feature to restrict AI-generated libraries to their owners
- New system setting `restrict_ai_library_to_owner` (default: enabled)
- Admin toggle in Settings → System with i18n support for all languages

### Automated Docker Builds
- Configured GitHub Actions workflow for automatic Docker image building
- Images automatically pushed to `ghcr.io/damienigg/aperture` on every commit
- Simplified deployment and updates for users

## Bug Fixes

### DeepInfra Chat Model Initialization
- Fixed context window handling issues with DeepInfra models
- Resolved token limit errors that prevented chat window initialization
- Enhanced error handling and fallback mechanisms for model compatibility

### Docker Build Process
- Resolved TypeScript compilation errors in Docker builds
- Fixed dynamic import issues that caused build failures
- Improved module resolution for ES modules in Docker environment

## Technical Improvements

### AI Provider Architecture
- Enhanced provider abstraction layer with better error handling
- Improved model metadata management with context window information
- Added comprehensive validation for AI provider configurations

### Development Workflow
- Added automated testing infrastructure for library restriction features
- Created detailed documentation for DeepInfra integration and troubleshooting
- Improved build process reliability with static imports over dynamic loading

## Configuration Changes

### New System Settings
- `restrict_ai_library_to_owner` (boolean, default: true) - Controls AI library access restrictions
- Enhanced AI provider configuration with per-function model selection

### Docker Configuration
- Updated GitHub Actions workflow for streamlined Docker image deployment
- Added proper error handling and logging for build processes
- Configured automatic package permission verification for GHCR

## Documentation Updates

- Updated README with DeepInfra provider information
- Added comprehensive DeepInfra chat model fix documentation
- Enhanced setup guide with multi-provider AI configuration instructions
- Updated tech stack information to reflect new capabilities

## Compatibility Notes

This release maintains full backward compatibility with existing installations. The new owner-only library access feature is enabled by default but can be disabled in system settings if needed.

## Upgrade Instructions

For existing installations:
1. Pull the latest Docker image: `docker-compose pull`
2. Restart the service: `docker-compose up -d`
3. Configure DeepInfra provider in Settings → AI if desired
4. Verify library restriction settings in Settings → System

No database migrations are required for this update.
