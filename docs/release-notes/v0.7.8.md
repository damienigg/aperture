# Aperture v0.7.8 Release Notes

This release is a **maintenance and quality release** focused on simplifying large modules, reducing duplicated logic, improving settings reliability, and refreshing the login experience. **Baseline:** follows **v0.7.7**.

---

## At a glance

| Theme | What shipped |
| --- | --- |
| **Web maintainability** | Split large **Browse**, **Top Picks**, and **User Settings** surfaces into focused hooks, cards, tab components, and utilities. |
| **Core/API maintainability** | Split the user profile API handler and series sync orchestration into focused modules while preserving public routes and exports. |
| **AI cleanup** | Extracted shared playlist text generation helpers and moved the AI provider/model capability catalog into typed registry/data modules. |
| **Discovery** | Extracted shared lazy-load queue behavior for streaming and TMDb genre discovery strips. |
| **Emby provider** | Simplified Emby-only helpers for fetch/error handling, request builders, item mappers, and item updates. Jellyfin behavior was intentionally left untouched. |
| **Settings reliability** | UserSettings save failures are surfaced through existing UI instead of silently failing for required preference saves. |
| **Login** | Refreshed the login screen to match the Aperture left-rail logo treatment and improved dark-theme input readability. |

---

## Summary for users

- **Browse** should behave the same, but the implementation is now split into smaller modules for movies, series, people, filters, presets, and tab UI.
- **Top Picks settings** should behave the same, with a cleaner internal structure for source previews, output settings, images, and auto-request options.
- **User Settings** keeps algorithm controls in one canonical location and avoids duplicate include-watched / dislike behavior controls.
- The **login screen** now uses the Aperture mark and wordmark styling from the app shell, with clearer field contrast in the dark theme.

---

## Summary for admins

### User Settings cleanup

- Duplicate **include watched** and **dislike behavior** controls were removed from the Preferences tab.
- The canonical controls remain in **User Settings → AI Algorithm**.
- Required save failures for relevant preferences now surface through user-visible error UI.

### Top Picks settings cleanup

- The admin Top Picks section is now composed from focused modules under `settings/topPicks`.
- Preview count fetching was consolidated so unused duplicate preview state no longer issues extra requests.
- Preview effects now depend only on the source/list/language fields that actually affect the preview.

### Browse cleanup

- Browse is now a thin shell for URL tab state and view-mode composition.
- Movie, series, and people data loading, infinite scroll, filter presets, filter chips, and tab UI are split into focused modules.

### API, sync, and media-provider cleanup

- `/api/users/:id/*` profile routes are registered through focused route modules with shared auth/SSE helpers.
- Series sync is split into constants, types, reconciliation, series batches, episode batches, orchestration, watch-history sync, and shared sync helpers.
- Emby helper extraction reduces repeated fetch/request/mapper/update code without adding Jellyfin parity work.

## Update instructions

### For Docker users

```bash
docker compose pull
docker compose up -d
```

No new migration step is required for this release beyond your usual deployment process.

### Operator checklist

1. Confirm the app reports **v0.7.8** in the left rail / setup wizard and `/health`.
2. Smoke test **login**, **User Settings**, **Top Picks settings**, and **Browse** after deploy.
3. If you use Emby, spot-check a critical provider flow such as library listing or playlist/collection updates.
